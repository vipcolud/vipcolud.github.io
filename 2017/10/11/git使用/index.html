git基本操作

发表于 2017-10-11 | 分类于 git

1.首先新建仓库 然后将其克隆下来
git clone https://github.com/vipcolud/vipcolud.git
2.然后把项目拷贝到该本地仓库
3.将项目加入
git add .

4.提交注释
git commit -m “3”

5.提交
git push

6.状态
git status
码云上面最好用powershell

  • git init 创建新仓库
  • git clone /path/to/repository 创建一个本地仓库的克隆版本
  • 你的本地仓库由 git 维护的三棵“树”组成。第一个是你的 工作目录，它持有实际文件；第二个是 缓存区（Index），它像个缓存区域，临时保存你的改动；最后是 HEAD，指向你最近一次提交后的结果。
  • git add git add * 添加到缓存区
  • git commit -m “代码提交信息” 提交改动 将代码提交到head中
  • git push origin master 将代码提交到远程仓库
  • git remote add origin 如果你还没有克隆现有仓库，并欲将你的仓库连接到某个远程服务器
  • 分支是用来将特性开发绝缘开来的。在你创建仓库的时候，master 是“默认的”。在其他分支上进行开发，完成后再将它们合并到主分支上。
  • git checkout -b feature_x 创建一个叫做”feature_x”的分支，并切换过去
  • git checkout master 切换回主分支
  • git branch -d feature_x 再把新建的分支删掉
  • git push origin 除非你将分支推送到远端仓库，不然该分支就是不为他人所见的
  • git pull 更新本地仓库至最新改动
  • git merge 合并其他分支到你的当前分支
  • git add 自动合并不一定每次都成功，此命令将其合并成功
  • git diff 查看改动
  • git tag 1.0.0 1b2e1d63ff 创建一个叫做1.0.0的标签
  • git log 获取提交ID，1b2e1d63ff 是标记的提交ID
  • git checkout – 替换掉本地改动
  • git fetch origin git reset –hard origin/master 丢弃本地改动与提交，获取服务器上最新的版本并将本地主分支指向它
  • gitk 图形化git
  • git config color.ui true 彩色的git输出
  • git config format.pretty oneline 显示历史记录时，只显示一行注释信息
  • git add -i 交互地添加文件至缓存区
